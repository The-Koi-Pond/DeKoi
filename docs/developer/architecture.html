Architecture

The current tree is layered by ownership, not by convenience.

DeKoi is a Tauri desktop app with a React feature tree, a React-free TypeScript product engine, typed frontend runtime adapters, embedded Tauri commands, a hostable Rust HTTP runtime, and focused Rust capability crates. The current feature tree is physically split into catalog, runtime, modes, and shell, with dependency direction enforced by .dependency-cruiser.cjs.

Layer Current packages Allowed role Must not do
features/catalog keepers, companions, bubble-presets, Bubbles, connections, gallery, knowledge, lorebooks, personas, presets, sprites. Own resource hooks, storage-facing types, query keys, and library/editor surfaces. Import runtime systems, mode UI, or shell tools.
features/runtime generation, ripples, visuals, ripples. Own shared runtime systems used by modes and shell. Import concrete modes or app-shell workflows.
features/modes Bubbles, VN, future-adventure, media-assets, router, shared/Bubbles-ui, shared/scene-ui. Own concrete user modes, shared mode UI, and the mode composition point. Import shell tools, or import sibling concrete modes outside modes/router.
features/shell bot-browser, connections, imports, pond-guide, notifications, onboarding, settings, spotify. Own app-level panels, modals, native integration UI, top-level reset/import workflows, and standalone guide. Own mode product rules or duplicate engine orchestration.
The enforced direction is shell -> modes -> runtime -> catalog. Same-layer callers should still use public owner files instead of reaching into another package's private folders.

modes/media-assets Managed game asset browser and editor UI. It is a mode-adjacent tool composed by the app shell through shell.ts.
features/shell/* Settings, provider connection management UI, admin data reset, guide, SillyTavern imports, bot browser, onboarding, notification bubbles, and Spotify controls. Shell may coordinate top-level workflows, but storage formats, prompt behavior, provider transport, and native IO still belong lower.

Dependency Guard

Guard What it enforces
No circular dependencies Module graphs must remain acyclic.
Shared stays lower src/shared cannot import src/features or src/app.
Engine stays framework and host independent src/engine cannot import React, Zustand, @tauri-apps/api, or src/shared/api.
Feature layers only flow downward catalog cannot import higher feature layers; runtime cannot import modes or shell; modes cannot import shell.
Concrete modes are separated Bubbles, VN, and game cannot import each other's concrete UI or engine mode internals. features/modes/router is the composition exception for React surfaces.
Shared mode UI stays lower than concrete modes features/modes/shared cannot import concrete Bubbles, VN, or future adventure surfaces packages.
Feature internals are private Cross-package imports into another package's components, hooks, stores, state, lib, api, or encounter folders are blocked.

Runtime Host Scope

The implemented hosts are the embedded Tauri desktop runtime and the hostable dekoi-server Rust runtime. The shared API layer owns the transport decision: when remoteRuntimeUrl is set, supported storage/import/connection/LLM calls route through HTTP; unsupported desktop-native actions stay on embedded Tauri and must not grow silent browser fallbacks.

When Adding Or Moving Code

  1. Start with the owner: catalog data, runtime system, concrete mode, shell workflow, shared UI/helper, engine service, shared API adapter, embedded Tauri command, hostable HTTP dispatch, or Rust capability.
  2. Use a package public entrypoint when crossing feature package boundaries.
  3. Move reusable mode UI to features/modes/shared; move reusable runtime systems to features/runtime; move generic UI to src/shared/components.
  4. Move product behavior to the owning engine mode or generation layer, not into React components.
  5. Move privileged local work to Rust commands/crates and expose a narrow shared API wrapper.
  6. For hostable runtime support, add the focused shared API wrapper first, then route only supported commands through remote-runtime.ts, http_server.rs, and http_dispatch.rs.
  7. Implement engine capability ports at the feature/app edge with shared API wrappers; do not import shared API adapters into engine modules.
  8. Run pnpm check:architecture when imports or package structure change, and pnpm check:docs when these docs change.

If a convenient import crosses ownership, add or adjust a public owner contract instead of reaching into a private folder.

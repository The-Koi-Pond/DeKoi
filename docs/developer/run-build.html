Local development

Run the current DeKoi seed with browser or desktop paths.

DeKoi runs as a Vite, React, and TypeScript app with an optional Tauri desktop host for native storage, bundle dialogs, provider-key secrets, and the desktop runtime bridge.

Runtime note: browser development is useful for UI work, but durable storage and saved provider keys require the Tauri desktop host or a compatible remote runtime.

Prerequisites

Tool Why it is needed
Node.js Runs Vite, TypeScript, and the package manager.
pnpm Installs dependencies and runs repo scripts.
Rust and Cargo Builds the optional Tauri desktop host.

Install Dependencies

pnpm install

Run The App

Start the Vite development server:

pnpm dev

Open the URL printed by Vite. By default this is usually http://localhost:5173/.

Run the desktop host when you need app-data storage, provider-key secrets, or native bundle dialogs:

pnpm tauri:dev

Desktop collection files are JSON arrays under <app-data>/collections. Malformed files and leftover recovery artifacts block normal autosave overwrite, and collection writes preserve a sibling .json.bak recovery file.

Run The Developer Docs

Serve this static documentation set:

pnpm docs:dev

Open http://127.0.0.1:4174/.

Build Check

Run the production frontend build:

pnpm build

Full Check

Run the local gate before shipping or handing off broad changes:

pnpm check

Lint Check

Run ESLint over source files:

pnpm lint

Current Manual QA

Messenger preview

Confirm the Pond rail, active Messenger thread, Companions, Lorebooks, and Ripple Dock all render.

Responsive layout

Check desktop and narrow widths. Text should remain readable and panels should stack cleanly.

Static model proof

Confirm empty storage starts without built-in companions, connections, lorebooks, personas, or threads.

Future Native Host Work

When Tauri is added, document its commands here only after the scripts exist in package.json and the native host has been verified locally.

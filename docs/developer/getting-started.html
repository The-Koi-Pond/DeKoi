Agent onboarding

Ask a coding agent “how do I get started?” and use this path.

This page gives coding agents and developers the same starting point: run the docs, learn the repo shape, run the current Vite app, test real workflows, then fix bugs from the correct owner.

Migration note: this page has been adapted for the fresh DeKoi seed. Tauri, hostable runtime, imports, and provider workflows are future validation targets, not current setup requirements.

First Agent Response

When asked how to get started, a coding agent should do this:

  1. Start the docs server with pnpm docs:dev.
  2. Give the user http://127.0.0.1:4174/.
  3. Teach the repo shape in a short layered tour.
  4. Show how to run the current Vite app.
  5. Guide a manual testing pass through the current Messenger preview.
  6. When a bug is found, switch into root-cause bug fixing.
pnpm docs:dev
pnpm install
pnpm dev
pnpm build

Manual Testing Path

Messenger

Create/select a connection, Companion, and Persona. Send a message, regenerate, retry, swipe/branch, test attachments and summaries.

Classic

Create, fork, and conclude a scene. Test choices, scene memory, sprites, and encounters.

Future adventure

Future adventure systems are parked until Messenger, Classic, native records, and storage are stable.

Pond Guide

Open Pond Guide from the shell, select a connection/persona, send text and file attachments, and ask a library question that should use the read-only creative library tool.

Settings and imports

Create provider connections, test them, import/export core objects, upload avatars/backgrounds/assets, and verify visible errors.

Remote runtime

Reserved for later. Do not test remote runtime behavior until DeKoi has a native host boundary.

Coding agents should not start coding from a vague onboarding request. They should run the docs, teach the repo, help the user test the app, then edit code only after there is a concrete bug or feature request.

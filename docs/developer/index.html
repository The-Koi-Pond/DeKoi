Start here

How this repo is put together and how to work in it without widening impact.

DeKoi is a React, TypeScript, Vite, and optional Tauri app. React features own the user interface, TypeScript engine modules own product behavior, the runtime bridge owns storage and import/export contracts, and the desktop host owns privileged capabilities.

Current status: these developer docs now describe the current DeKoi source lanes. They are not exhaustive product specs; check the source tree before planning work outside Messenger, Roleplay, catalog, storage, runtime, shell, or desktop host paths.

Current Tech Stack

Area Stack Current refactor role
Webview UI React 19, TypeScript 6, Vite 8. src/features owns the first visible surface. src/engine owns the first React-free product records.
Client state and data loading React state plus app-level storage sync. src/app owns top-level state, storage sync, and view action assembly.
Product engine React-free TypeScript records, pure actions, and generation request builders. src/engine currently defines app settings, Messenger, Roleplay, catalog records, generation, surfaces, and Ripples.
Desktop, local, and hostable capabilities Optional Tauri desktop host plus compatible remote runtime commands. Desktop storage, native bundle dialogs, provider secrets, provider transport, and runtime dispatch stay behind explicit command wrappers.

Refactor Coverage

Shell and feature split

The app shell lives under src/app. Product surfaces live under src/features, including shell, navigation, catalog, Messenger, Roleplay, and feature-runtime workflows.

Engine split

Engine code is intentionally flat for now: records, pure actions, generation helpers, and mode-specific request builders live in focused files.

Storage split

Storage contracts, collection adapters, split transcripts, bundle import/export, and legacy import live under src/runtime/storage.

Native capability split

src-tauri/src has focused modules for storage, provider secrets, file dialogs, host status, and desktop runtime dispatch.

Rules That Keep Changes Small

One owner

Every behavior belongs to React UI, the TypeScript engine, or Rust capability code. Do not duplicate ownership across layers.

Mode separation

Messenger is the first polish target. Roleplay has a working first slice. Reserved, media, presets, and Ripples editor surfaces are not complete product loops yet.

Explicit imports

Import owner modules and public feature APIs. Avoid deep imports into another feature's private files, and keep embedded Tauri or hostable HTTP calls behind shared API wrappers.

Root fixes

Fix broken contracts or owners. Do not add fake success paths, broad fallbacks, or UI-only patches.

Before a change: identify the owner, expected impact area, dependent callers, and checks. If a shared layer changes, verify every mode or feature that depends on that layer.

How this repo is put together and how to work in it without widening impact.

DeKoi is currently a fresh React, TypeScript, and Vite seed. These migrated developer docs preserve the intended layered architecture while the new implementation catches up: React owns the user interface, TypeScript engine modules own product behavior, and future host layers should own privileged capabilities.

Migration note: these docs were ported from user-owned developer documentation and rewritten with DeKoi terms. Treat deeper architecture pages as a validation queue until the new codebase implements the referenced layers.

Current Tech Stack

Area Stack Current refactor role
Webview UI React 19, TypeScript 6, Vite 8. src/features owns the first visible surface. src/engine owns the first React-free product records.
Client state and data loading React state first. Dedicated data loading and app stores are not selected yet. The first slice should prove Messenger locally before adding provider, storage, or runtime complexity.
Product engine React-free TypeScript model modules, then storage schemas after the names settle. src/engine currently defines Messenger, Companions, Personas, Lorebooks, and Ripples.
Desktop, local, and hostable capabilities Future Tauri/native capabilities. Native storage, secrets, filesystem access, provider transport, and imports should stay behind explicit runtime boundaries when they are added.

Refactor Coverage

Shell and feature split

The app shell lives under src/app. Product surfaces live under src/features, including Messenger, Classic, future adventure systems, media assets, libraries, settings, integrations, onboarding, and Pond Guide.

Engine split

Engine code is separated into contracts, capabilities, pure helpers, repositories, generation building blocks, helper runtime, generation lifecycle, standalone guide entry, and mode-owned services.

Pond Guide path

Pond Guide is not hidden inside Messenger. The shell opens features/shell/pond-guide, which calls engine/pond-guide, shared/api/pond-guide-api, and the Rust pond_guide_prompt command.

Native capability split

src-tauri is split into thin command facades plus focused crates for core helpers, security, storage, assets, LLM transport, and integrations. The same focused Rust modules back the embedded Tauri commands and hostable HTTP dispatch.

Rules That Keep Changes Small

One owner

Every behavior belongs to React UI, the TypeScript engine, or Rust capability code. Do not duplicate ownership across layers.

Mode separation

Messenger is the first active surface. Classic is reserved next. Future adventure systems should stay separate until DeKoi has native records and storage.

Explicit imports

Import owner modules and public feature APIs. Avoid deep imports into another feature's private files, and keep embedded Tauri or hostable HTTP calls behind shared API wrappers.

Root fixes

Fix broken contracts or owners. Do not add fake success paths, broad fallbacks, or UI-only patches.

Before a change: identify the owner, expected impact area, dependent callers, and checks. If a shared layer changes, verify every mode or feature that depends on that layer.
